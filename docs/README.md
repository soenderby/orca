# Orca Documentation Layout

This directory separates **future planning** from **current runtime functionality**.

## Boundaries

- Current operational docs stay at repo root:
  - `README.md`
  - `AGENT_PROMPT.md`
  - `OPERATOR_GUIDE.md`
- Planning and redesign material lives in `docs/planning/`.
- External/reference research material lives in `docs/research/`.

Treat files under `docs/planning/` as design input, not runtime source of truth.
